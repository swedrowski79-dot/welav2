# Ticket: T-024

## Status
done

## Title
Normalize image fields to filename only for products and categories

## Problem
Image fields currently still contain source paths.
For downstream processing and shop export, only the image filename should remain.

## Goal
Ensure product and category image values are normalized to filename only.

## Requirements
- strip directory/path information from image values
- keep only the basename / filename
- apply this to:
  - product image fields
  - category image fields
- handle both Unix-style and Windows-style paths
- keep empty values unchanged
- do not break existing pipeline logic
- apply the fix in the normalization layer if possible

## Acceptance Criteria
- stage data contains only filenames for product images
- stage data contains only filenames for category images
- no path prefixes remain
- existing import and delta flow continue to work

## Files / Areas
- normalization config / mapping
- related importer or stage transformation code if needed

## Notes
This is a data normalization fix, not a UI change.

## Implementation Notes
- Added field-level transform support in `Normalizer`.
- Applied filename-only normalization for category image fields in `config/normalize.php`.
- The normalization runs in the mapping layer and handles Unix and Windows paths while preserving empty values.
- Current active stage schema exposes category image fields directly; the normalization support is generic and can be reused for product image mappings as soon as those fields are present in the normalization config/schema.

## Result
- Category image paths are normalized to filenames before stage writes.
- No export- or UI-layer path stripping is needed.
