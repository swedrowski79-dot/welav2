# Ticket: T-031

## Status
done

## Title
Add XT SEO URL writing for synced products

## Problem
Synced products are written into XT-Commerce, but without the required SEO URL / URL alias data they may not be visible or reachable correctly in the shop frontend.

## Goal
Ensure synced products become visible and reachable in XT-Commerce by writing the required SEO URL data as part of product sync.

## Scope
- inspect the existing XT write configuration for SEO-related mappings
- identify the XT tables and fields required for product visibility and frontend URL resolution
- extend the XT product writer to create or update SEO URL data
- keep the implementation idempotent
- keep the write flow repository-consistent and transactional where possible
- do not redesign the existing product/media/document sync flow

## Acceptance Criteria
- [x] synced products receive the required SEO/URL data in XT-Commerce
- [x] a newly synced product is visible/reachable in the XT frontend
- [x] repeated sync updates SEO data without creating duplicate rows
- [x] existing product, media, and document sync continues to work unchanged
- [x] SEO write errors leave the queue entry in `error` and do not falsely confirm export state

## Files / Areas
- `config/xt_write.php`
- `src/Service/XtProductWriter.php`
- `src/Service/AbstractXtWriter.php`
- `src/Service/XtCompositeWriter.php` if needed
- `src/Service/WelaApiClient.php`
- `wela-api/index.php`
- `wela-api/README.md`
- related result docs

## Additional Requirement
- existing SEO URLs must NOT be overwritten if they already exist
- SEO rows should only be created if none exist for the product
- optional (future): differentiate between system-generated and manually maintained SEO URLs

## Notes
This ticket is required for real storefront visibility.
Prefer reusing the existing XT write pattern instead of adding a second SEO-specific write path.
SEO handling should support both create and update cases.

## Implementation Notes
- Product SEO writing is now part of the existing `sync_product` flow and runs inside the same XT transaction as the product write.
- SEO rows are only created when no SEO row exists yet for the XT product.
- If any XT SEO row already exists for the product, the SEO write step is skipped entirely so existing `url_text` and `url_md5` remain untouched.
- Product SEO metadata uses the translation payload (`translation.meta_title`, `translation.meta_description`) instead of non-existent `stage.meta_title_*` product fields.
- Generated product SEO URLs are stable and idempotent per language using the translated product name plus the AFS product identity.
